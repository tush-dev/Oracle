# Contributing to Oracle

Thank you for contributing to Oracle. This guide explains how to make improvements in a way that keeps the project consistent.

## Recommended workflow

1. Create a new branch for your change.
2. Make a small, focused edit.
3. Use a descriptive commit message.
4. Keep runtime behavior unchanged when updating documentation or comments.

## Commit message style

A good commit message should:

- describe what changed
- explain why the change was made
- avoid overly broad messages like "update"

Example:

- `docs: add backend env example`
- `chore: document backend setup steps`
- `refactor: add comments to GitHub OAuth route`

## Documentation updates

- Use `README.md` files for project-level documentation.
- Add environment examples to `.env.example` files.
- Keep docs current with the repository structure.
